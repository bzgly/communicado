# Automated Closed Loop: оптимизация коммуникации Банка России

Closed-loop система с синтетическими интервью: от пресс-релиза до персонального объяснения каждому гражданину.

## Проблема

ЦБ публикует один пресс-релиз на языке экспертов. Эксперт и пенсионер читают одну фразу -- и реагируют **противоположно** (Kim & Lee, "Bounded Rationality in CB Communication").

| Факт | Источник |
|------|----------|
| 30% домохозяйств обновляют ожидания после решений ЦБ | Lamla & Vinogradov (2024, JME) |
| Flesch Reading Ease оригинальных релизов -- 35 ("очень сложный") | Оборнева (2006) |
| Sentiment: тревожных слов больше успокаивающих | -- |
| 37% россиян узнают новости из Telegram | НАФИ (2024) |

## Архитектура

Текст подается на вход, агенты интервьюируют, анализируют, переписывают, валидируют -- и цикл повторяется.

```
Релиз ЦБ -> Экстрактор (5 тезисов) -> Интервью (6 персон x 6 вопросов) -> Аналитик (Score + проблемы) -> Редактор -> Валидатор (сохранность тезисов) -> [повтор]
```

### Три системы метрик

- **Communication Score** (LLM, 0-100) -- оценивает понятность с anchor points калибровки
- **Flesch Reading Ease** -- детерминированная метрика читаемости для русского языка (Оборнева, 2006)
- **Sentiment Balance** -- словарный баланс тревожных vs успокаивающих слов, не зависит от LLM

## Персоны

6 синтетических персон, 19 полей, 7 осей дифференциации из литературы. Формат профилей воспроизводит подход СФГ Банка России.

| Персона | Роль | Ключевая ось | Attention |
|---------|------|-------------|-----------|
| Валентина, 67 | Пенсионерка, Воронеж | Низкая грамотность, ТВ, высокая тревожность | ~25% |
| Алексей, 38 | Финдиректор, Москва | Высокая грамотность, cbr.ru напрямую | ~95% |
| Ольга, 34 | Мама с ипотекой, Казань | Максимальная чувствительность к ставке | ~60% |
| Сергей, 45 | Водитель, Челябинск | Мин. активы, мин. внимание к ЦБ | ~15% |
| Павел, 51 | Владелец бизнеса, Краснодар | Кредитный канал, pricing power | ~30% |
| Даниил, 22 | Студент-экономист, СПб | Молодежь, соцсети, теория без практики | ~15% |

## Результаты

На 4 реальных пресс-релизах ЦБ (Окт 2025 -- Мар 2026), 6 персон, adaptive-интервью:

- **4/4** прогона показывают стабильный рост Score
- **+27** средний прирост Communication Score (от 35-42 до 62-72)
- **0.0** sentiment после оптимизации -- тревога нейтрализована
- **72** лучший Score (релиз 20.03.2026)

### До / После (релиз 20.03.2026, снижение ставки до 15%)

| Оригинал (Score 42) | После 3 итераций (Score 72) |
|---------------------|----------------------------|
| "снизить ключевую ставку на 50 б.п., до 15,00% годовых" | "снизить ключевую ставку на 0,5 процентного пункта -- до 15%. От этой ставки зависят проценты по вкладам и кредитам" |
| "устойчивые показатели текущего роста цен остаются в диапазоне 4-5% в пересчете на год" | "базовая инфляция -- рост цен на товары и услуги, кроме продуктов и топлива -- остается в диапазоне 4-5% в год" |
| "значимо выросла неопределенность со стороны внешних условий" | "ситуация в мире стала менее предсказуемой, и это может повлиять на российскую экономику" |

## Каналы

Из одного релиза -- текст для любого канала. Каждый канал оптимизируется через свой closed loop с персонами целевой аудитории.

| Канал | Аудитория | Score | Flesch | Стиль |
|-------|-----------|-------|--------|-------|
| Пресс-релиз cbr.ru | Алексей, Павел | 45 -> **68** | 35 -> 23 | Официальный, с терминами |
| ТГ-канал ЦБ | Алексей, Даниил, Ольга | 45 -> **78** | 35 -> 24 | Доступный, до 1000 знаков |
| Деловые СМИ | Алексей, Павел | 45 -> **75** | 35 -> 14 | Цифры, прогнозы, кредиты |
| VK Клипы / Reels | Даниил, Ольга | 45 -> **78** | 35 -> 49 | Сценарий 60 сек, на "ты" |

## Telegram-бот (@centralbanbot)

Персональная коммуникация: бот узнает пользователя -> строит персону -> прогоняет closed loop -> выдает персональный пресс-релиз + отвечает на вопросы.

1. **Знакомство** -- бот расспрашивает: возраст, профессия, ипотека, вклады, откуда новости
2. **Построение персоны** -- из ответов строится профиль с 19 полями
3. **Closed Loop** -- пресс-релиз прогоняется через цикл оптимизации для этого пользователя
4. **Персональный текст + Q&A** -- пользователь получает свое объяснение и может задать вопросы

## Структура проекта

```
excommunicado/
  engine.py          -- closed-loop: extract -> interview -> analyze -> edit -> validate -> repeat
  interview.py       -- LLM-интервьюер: scripted (6 фиксированных вопросов) и adaptive (LLM follow-up)
  personas.py        -- 6 синтетических персон с 19 полями, обоснованных из литературы
  prompts.py         -- системные промпты для всех агентов
  readability.py     -- детерминированные метрики: Flesch, sentiment, jargon count
  channels.py        -- 5 каналов коммуникации ЦБ с целевыми аудиториями
  sample_releases.py -- 4 реальных пресс-релиза ЦБ (Окт 2025 -- Мар 2026)
  mock_data.py       -- реалистичные мок-данные для демо без API
  bot.py             -- Telegram-бот персональной коммуникации
  app.py             -- Streamlit-демо (closed loop + каналы + сравнение)
  run_one.py         -- запуск одного прогона (релиз или канал)
  run_benchmarks.py  -- массовые прогоны для презентации
  build_charts.py    -- графики из результатов (Plotly)
  build_presentation.py -- генератор PDF-презентации (reportlab)
  test_prod.py       -- прод-тесты с реальным LLM
  speech.md          -- речь к защите (5-7 минут)
  presentation.pdf   -- презентация
  requirements.txt   -- зависимости
  tests/             -- юнит-тесты (персоны, промпты, читаемость, каналы, интеграция)
```

## Запуск

```bash
pip install -r requirements.txt

# Streamlit-демо
streamlit run app.py

# Telegram-бот (нужен BOT_TOKEN в .env)
python bot.py

# Один прогон
python run_one.py release 3       # последний релиз, 3 итерации
python run_one.py channel vk_clips

# Тесты
python -m pytest tests/ -v
```

Для работы с LLM нужны переменные окружения `SOY_TOKEN` и `ANTHROPIC_KEY` (или `USE_MOCK=1` для работы без API).

## Ограничения

- **LLM склонны к стереотипизации** -- персоны откалиброваны по 7 осям из литературы; следующий шаг: валидация на реальных транскриптах СФГ
- **Score может колебаться** -- поэтому добавлены детерминированные метрики (Flesch, jargon count, sentiment balance)
- **Не замена фокус-группы** -- инструмент быстрых итераций между реальными фокус-группами (минуты вместо недель)
- **Упрощение может усиливать тревогу** -- эффект Kim & Lee: понятный текст доносит тревожный сигнал сильнее
